## Статус структуры
Текущая структура проекта считается готовой для старта разработки.

Основа:
- `RPD.md` как главный продуктовый документ.
- MVP-архитектура: React + Vite + TypeScript + Tailwind + Framer Motion + Supabase.
- Хостинг: GitHub + Vercel (или аналог бесплатного тарифа).

## Этапы разработки (1 задача = 1 пункт)

1. Инициализация репозитория и проекта (`Vite + React + TypeScript`, Tailwind, Framer Motion, Lucide).
2. Настройка окружения (`.env`, `.env.example`, Supabase client).
3. Создание схемы БД (`students`, `badges`, `student_badges`) и ограничений (`score >= 0`).
4. Настройка Auth + RLS (публичное чтение, запись только teacher по `auth.uid()`), отключение публичной регистрации и создание teacher-аккаунта в Supabase Dashboard.
5. Добавление seed-данных (бейджи и тестовые ученики).
6. Реализация публичного leaderboard (сортировка, карточки, Top-3 стили, mobile-first).
7. Подключение Realtime-обновлений из Supabase (включая включение репликации таблиц `students` и `student_badges` в Supabase Dashboard -> Replication).
8. Добавление анимаций изменения позиций (Framer Motion).
9. Реализация входа в админку и защищенных роутов.
10. Реализация управления учениками и баллами (`+1`, `+5`, `-1`, add/remove student).
11. Реализация назначения и снятия бейджей.
12. Добавление loading/empty/error состояний и уведомлений.
13. Проверка адаптивности и базовой доступности на мобильных устройствах.
14. Проверка всех ключевых сценариев по Definition of Done.
15. Обновление `README.md` (запуск, env, Supabase setup, деплой).
16. Подключение GitHub -> Vercel и проверка автодеплоя из `main`.
17. Финальная приемка и фиксация готовности MVP.
