# RPD v2: MasterRank (pet-project)

## 1. Цель проекта
MasterRank - это веб-приложение для рейтинга учеников в реальном времени.

Пользовательские роли:
- Ученики: смотрят текущий рейтинг.
- Учитель: управляет учениками, баллами и бейджами через админ-панель.

Принципы проекта:
- Простая реализация без лишней архитектурной сложности.
- Бесплатный хостинг и простой деплой.
- Приоритет мобильному интерфейсу.

## 2. Финальная архитектура MVP

### Frontend
- React + Vite + TypeScript
- Tailwind CSS
- Framer Motion
- Lucide React

### Backend и данные
- Supabase:
  - PostgreSQL
  - Realtime subscriptions
  - Auth
  - RLS (Row Level Security)

### Инфраструктура
- GitHub репозиторий как source of truth.
- Автодеплой на Vercel (или аналогичном бесплатном хостинге) из ветки `main`.

Почему так: этот стек позволяет быстро сделать рабочий продукт без отдельного backend-сервера и без затрат на инфраструктуру.

## 3. UI/UX требования
- Мобильный-first интерфейс.
- Темная тема по умолчанию.
- Крупный, читаемый текст.
- Минималистичный стиль без перегруженных элементов.
- Карточки учеников с понятными иконками бейджей.

Top-3 должны визуально выделяться:
- 1 место: золотой стиль
- 2 место: серебряный стиль
- 3 место: бронзовый стиль

## 4. Функциональные требования (MVP)

### 4.1 Публичная часть (Leaderboard)
- Список учеников, отсортированный по убыванию баллов.
- На карточке: имя, текущий балл, набор бейджей.
- Real-time обновление без перезагрузки страницы.

### 4.2 Админ-панель (Teacher Admin)
- Вход только для учителя через Supabase Auth (email/password).
- Управление баллами быстрыми действиями: `+1`, `+5`, `-1`.
- Назначение/снятие бейджей.
- Добавление и удаление учеников.

## 5. Бизнес-правила
- `score >= 0` (балл не может быть отрицательным).
- Сортировка: сначала `score DESC`, при равенстве `name ASC`.
- Операции по изменению данных доступны только учителю.
- Публичным пользователям доступно только чтение рейтинга.

## 6. Структура базы данных (Supabase)

### Таблица `students`
- `id uuid primary key`
- `name text not null`
- `score int not null default 0 check (score >= 0)`
- `created_at timestamptz not null default now()`

### Таблица `badges`
- `id uuid primary key`
- `code text unique not null`
- `title text not null`
- `icon text not null` (каноническое имя иконки из Lucide, например `star`, `trophy`)
- `created_at timestamptz not null default now()`

### Таблица `student_badges`
- `student_id uuid not null references students(id) on delete cascade`
- `badge_id uuid not null references badges(id) on delete cascade`
- `assigned_at timestamptz not null default now()`
- `primary key (student_id, badge_id)`

Почему не `jsonb`: отдельные таблицы для бейджей проще фильтровать, проверять и поддерживать даже в маленьком проекте.

## 7. Минимальная модель безопасности
- Используем Supabase Auth для teacher-аккаунта.
- Публичную регистрацию в Supabase Auth отключаем (`Enable email signups = OFF`), чтобы новые пользователи не могли получить роль `authenticated`.
- Создаем один teacher-аккаунт вручную в Supabase Dashboard (Authentication -> Users) и фиксируем его `auth.uid()`.
- RLS-политики:
  - `students`, `badges`, `student_badges`: `SELECT` разрешен всем (публичный leaderboard).
  - `INSERT/UPDATE/DELETE`: только учителю по проверке `auth.uid() = '<teacher_uuid>'`.
- Скрытый URL можно использовать только как дополнительный слой удобства, но не как защиту.

## 8. Развертывание и окружение
- Репозиторий хранится в GitHub.
- Деплой через интеграцию GitHub -> Vercel.
- Каждый push в `main` запускает production deploy.

Обязательные env-переменные фронтенда:
- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_ANON_KEY`

Для pet-проекта на старте допускается один Supabase-проект без отдельного `dev/prod` разделения.

## 9. Definition of Done (MVP)
- Leaderboard отображает учеников и корректно сортирует позиции.
- Изменения в баллах/бейджах отображаются на клиенте в реальном времени.
- Учитель может:
  - войти в админку,
  - добавить/удалить ученика,
  - изменить баллы,
  - назначить/снять бейдж.
- Неавторизованный пользователь не может изменять данные.
- Интерфейс стабильно работает на мобильных устройствах.

## 10. Порядок реализации

5. Подключить Realtime обновление.
6. Реализовать простую админ-панель с авторизацией учителя.

8. Проверить Definition of Done.
